# Writing systems & scripts

## Definition

Writing systems and scripts are the sets of visual signs, and the rules for combining them, that record the world's languages in visible form.

## Why it matters

The script, not the country or the language, is the real unit of support in product and type work. One script can write many languages and one language can be written in several scripts, so "we support Hindi" is really a question about whether your fonts, shaping, and layout handle the script behind it. Scripts also differ in kind, not just in letters: some give every sound its own letter, some attach vowels to consonants, some pack syllables into blocks or stack them vertically, and some run top to bottom. Knowing how a script is classified tells you what work it needs, which spacing and line-height assumptions break, and where the Latin defaults baked into most tools quietly fail. This category gives you the vocabulary to plan that work deliberately and to talk about it precisely with the people who speak and write these languages.

## Where to learn about more scripts

This glossary covers a representative spine of scripts, not every script in use. Many writing systems are deliberately out of scope for now (Khmer, Lao, Burmese, Braille, N'Ko, and the long tail), and will come through later cohorts or open-source contribution. To research a script the glossary does not yet cover, start with the three authority hubs the project itself relies on: [ScriptSource](https://scriptsource.org/) for writing-systems detail, [Glottolog](https://glottolog.org/) for languages and their relationships, and the [Unicode Standard](https://www.unicode.org/standard/standard.html) for how a script is encoded.
